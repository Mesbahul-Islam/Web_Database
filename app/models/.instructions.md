# Model Standards — Puutarhakanta API

This file applies to all files in `app/models/`.

## Model File Structure

Every model file should follow this pattern:

```python
from .base import Base
from typing import Optional
from datetime import datetime
from sqlalchemy import Column, ForeignKeyConstraint, Index, String, Table, Text, text
from sqlalchemy.dialects.mysql import INTEGER, TINYINT
from sqlalchemy.orm import Mapped, mapped_column, relationship


class <Table>(Base):
    __tablename__ = '<table_name_in_sql>'
    __table_args__ = (
        ForeignKeyConstraint(['fk_column'], ['referenced_table.referenced_column'], name='constraint_name'),
        Index('index_name', 'column1', 'column2'),
    )
    
    # Primary key
    id: Mapped[int] = mapped_column(INTEGER(11), primary_key=True)
    
    # Required columns
    name: Mapped[str] = mapped_column(String(255), nullable=False, server_default=text("''"))
    
    # Optional columns
    description: Mapped[Optional[str]] = mapped_column(Text)
    
    # FK columns (required)
    parent_id: Mapped[int] = mapped_column(INTEGER(11), nullable=False, server_default=text('0'))
    
    # FK columns (optional)
    ref_id: Mapped[Optional[int]] = mapped_column(INTEGER(11))
    
    # Relationships
    parent: Relationship["Parent"] = relationship("Parent", back_populates="children")
```

## Column Mapping Rules

### Use Mapped[] Type Hints (SQLAlchemy 2.x Style)

✅ **Correct**:
```python
name: Mapped[str] = mapped_column(String(255), nullable=False)
count: Mapped[Optional[int]] = mapped_column(INTEGER(11))
```

❌ **Wrong**:
```python
name = Column(String(255), nullable=False)
count = Column(INTEGER(11))
```

### Required Columns

- Use `Mapped[<type>]` (no Optional)
- Always include `nullable=False`
- Use `server_default=text("''")` for string defaults or appropriate types for others

```python
name: Mapped[str] = mapped_column(String(255), nullable=False, server_default=text("''"))
status: Mapped[int] = mapped_column(INTEGER(11), nullable=False, server_default=text('1'))
```

### Optional Columns

- Use `Mapped[Optional[<type>]]`
- Omit `nullable` (defaults to True)
- Omit `server_default` unless specified in schema

```python
description: Mapped[Optional[str]] = mapped_column(Text)
notes: Mapped[Optional[int]] = mapped_column(INTEGER(11))
```

### Foreign Key Columns (Required)

- Use `nullable=False, server_default=text('0')` (MySQL pattern)
- Use `ForeignKeyConstraint` in `__table_args__`, NOT inline `ForeignKey()`

```python
lahettaja_id: Mapped[int] = mapped_column(INTEGER(11), nullable=False, server_default=text('0'))

__table_args__ = (
    ForeignKeyConstraint(['lahettaja_id'], ['lahettaja.lahettaja_id'], name='fk_lahettaja'),
)
```

### Foreign Key Columns (Optional)

- Use `Mapped[Optional[int]]`
- Use `ForeignKeyConstraint` in `__table_args__`

```python
viite_nro: Mapped[Optional[int]] = mapped_column(INTEGER(11))

__table_args__ = (
    ForeignKeyConstraint(['viite_nro'], ['viite.viitenro'], name='fk_viite'),
)
```

## Relationship Declarations

Always declare relationships with `back_populates` on BOTH sides:

```python
# In Parent model:
children: Relationship[list["Child"]] = relationship("Child", back_populates="parent")

# In Child model:
parent: Relationship["Parent"] = relationship("Parent", back_populates="children")
```

### Use Forward References

Relationships to other models use string quotes to avoid import order issues:

```python
heimo: Relationship["Heimo"] = relationship("Heimo", back_populates="taksoni_list")
```

## Primary Key Definition

- Usually named `<table>_id` or `<table>_nro` (follow SQL schema)
- Always `INTEGER(11)` with `primary_key=True`

```python
taksonin_nro: Mapped[int] = mapped_column(INTEGER(11), primary_key=True)
lahettaja_id: Mapped[int] = mapped_column(INTEGER(11), primary_key=True)
```

### Special Case: Composite Primary Keys

If a table has multiple PK columns, use `__mapper_args__`:

```python
class ListaTarkastajanimi(Base):
    __tablename__ = 'lista_tarkastajanimi'
    __mapper_args__ = {"primary_key": [id, nimi]}
    
    id: Mapped[int] = mapped_column(INTEGER(11))
    nimi: Mapped[str] = mapped_column(String(255))
```

## Imports

Every model file should have these imports (adjust as needed):

```python
from .base import Base
from typing import Optional
from datetime import datetime
from sqlalchemy import ForeignKeyConstraint, Index, String, Text, text
from sqlalchemy.dialects.mysql import INTEGER, TINYINT
from sqlalchemy.orm import Mapped, mapped_column, relationship, Relationship
```

### Import Aliases

- `Mapped` and `mapped_column` from `sqlalchemy.orm`
- `Relationship` from `sqlalchemy.orm` (for type hints)
- `relationship` from `sqlalchemy.orm` (for declarations)
- MySQL-specific types from `sqlalchemy.dialects.mysql`

## __table_args__ Order

Place in this order:
1. ForeignKeyConstraint entries (first)
2. Index entries (second)
3. Other constraints (rare)

```python
__table_args__ = (
    ForeignKeyConstraint(['parent_id'], ['parent.id'], name='fk_parent'),
    ForeignKeyConstraint(['ref_id'], ['ref.ref_id'], name='fk_ref'),
    Index('idx_name', 'name'),
    Index('idx_created', 'created_at'),
)
```

## Common Column Types

| SQL Type | Python Type | SQLAlchemy |
|----------|-------------|-----------|
| INT(11) | int | INTEGER(11) |
| VARCHAR(255) | str | String(255) |
| TEXT | str | Text |
| DATE | datetime.date | Date |
| DATETIME | datetime.datetime | DateTime |
| TINYINT | int | TINYINT |
| DECIMAL(10,2) | Decimal | DECIMAL(10, 2) |

## Never

- ❌ Use inline `ForeignKey()` in columns
- ❌ Mix `Column()` and `mapped_column()` in same file
- ❌ Forget `server_default` on required FK columns
- ❌ Use `Mapped[Optional[str]]` when schema says `NOT NULL`
- ❌ Declare relationships without `back_populates`
- ❌ Use bare string literals in relationships (always use forward references)
- ❌ Modify `generated_models.py` — it's reference only

## Schema Source of Truth

Always check `puutarha_sqlite.sql` for:
- Exact column names and types
- Whether columns are nullable
- Default values
- Foreign key constraints and their names
- Indexes

Use that as the source of truth, not the SQL dump or legacy schema.
