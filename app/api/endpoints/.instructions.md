# Endpoint Standards — Puutarhakanta API

This file applies to all files in `app/api/endpoints/`.

## Endpoint File Structure (Strict Pattern)

Every endpoint file must follow this exact 23-line structure. No deviations.

```python
from fastapi import APIRouter, Depends, HTTPException, Request
from sqlalchemy.orm import Session
from typing import List

from app.api.query import apply_filters
from app.database import get_db
from app.models.<table> import <Model> as Model
from app.schemas.<table> import <Schema> as Schema

router = APIRouter()

@router.get("/", response_model=List[Schema])
def read_all(request: Request, skip: int = 0, limit: int = 100, db: Session = Depends(get_db)):
    query = apply_filters(db.query(Model), Model, request.query_params)
    return query.offset(skip).limit(limit).all()

@router.get("/{pk}", response_model=Schema)
def read_one(pk: str, db: Session = Depends(get_db)):
    item = db.query(Model).filter(Model.<pk_column> == pk).first()
    if not item:
        raise HTTPException(status_code=404, detail="Not found")
    return item
```

## Naming Conventions

- **Router variable**: Always `router = APIRouter()`
- **Function names**: `read_all` and `read_one` (lowercase, underscore-separated)
- **Parameter names**: `pk` for primary key, `skip`/`limit` for pagination
- **Model import alias**: `as Model` (capitalized)
- **Schema import alias**: `as Schema` (capitalized)

## Query Parameter Handling

- All filtering happens via `request.query_params`
- `apply_filters()` handles type coercion automatically
- `skip` and `limit` are reserved and handled separately
- Multi-value parameters are automatically supported by `apply_filters()`

## Response Models

- List endpoint: `response_model=List[Schema]`
- Single endpoint: `response_model=Schema`
- Pagination: Always `skip=0, limit=100` defaults
- 404: Return `HTTPException(status_code=404, detail="Not found")` when item not found

## Primary Key Column Names

Common PK column patterns (check schema for each table):
- `<table>_id` (e.g., `lahettaja_id` in `lahettaja` table)
- `<table>_nro` (e.g., `taksonin_nro` in `taksoni` table)
- `<table>_no` (rare)

Always match the actual column name from the model class.

## Database Session Management

- Session dependency: `db: Session = Depends(get_db)` (always included)
- Do NOT manage sessions manually
- Do NOT close sessions explicitly
- The `get_db()` dependency handles setup and cleanup

## Never

- Add business logic to endpoints
- Use custom filter implementations (use `apply_filters()` only)
- Add POST/PUT/DELETE routes
- Modify the SQLAlchemy query after `apply_filters()` (except `.offset()` and `.limit()`)
- Add authentication/authorization logic
- Cache results
- Add logging to endpoints

## If You Need to Create a New Endpoint

1. Verify the model exists in `app/models/<table>.py`
2. Verify the schema exists in `app/schemas/<table>.py`
3. Create `app/api/endpoints/<table>.py` with the standard pattern
4. Import `<table>` router in `app/api/api.py` (if not already done)
5. Register with `api_router.include_router(router, prefix="/<table>", tags=["<table>"])`
6. Do NOT register if the table is in the intentional exclude list (legacy/test tables)

## Query Parameter Examples

Valid requests:
- `GET /?skip=0&limit=50` — pagination
- `GET /?suku=Rosa` — single filter
- `GET /?suku=Rosa&lajin_auktori=Smith` — multiple filters
- `GET /?viitenro=123` — filter by FK column

Invalid requests (handled gracefully):
- `GET /?unknown_column=value` — unknown column ignored
- `GET /?skip=abc` — malformed, returns raw value, may cause DB error
- `GET /?limit=1000` — no upper limit enforced (allow large limits)
