# Schema Standards — Puutarhakanta API

This file applies to all files in `app/schemas/`.

## Schema File Structure (Three-Class Pattern)

Every schema file must have exactly three classes:

```python
from pydantic import BaseModel, ConfigDict
from typing import Optional, List


class <Table>Base(BaseModel):
    """Base schema with common fields for both Create and Read."""
    name: str
    description: Optional[str] = None
    count: int


class <Table>Create(<Table>Base):
    """Schema for POST/PUT operations (not used in read-only API, but kept for consistency)."""
    pass


class <Table>(<Table>Base):
    """Complete schema with from_attributes=True for ORM serialization."""
    model_config = ConfigDict(from_attributes=True)
    id: int
```

## The Three Classes Explained

### 1. `<Table>Base`

- Contains fields shared between Create and Read operations
- Does NOT include the primary key or computed fields
- Does NOT include the `model_config` line

```python
class TaksoniBas(BaseModel):
    tieteellinen_nimi: str
    suku: Optional[str] = None
    laji: Optional[str] = None
```

### 2. `<Table>Create`

- Inherits from `<Table>Base`
- Allowed on POST/PUT operations (not used in read-only API)
- Usually just `pass` in this codebase (kept for architectural consistency)

```python
class TaksoniCreate(TaksoniBase):
    pass
```

### 3. `<Table>` (Final Class)

- Inherits from `<Table>Base`
- Includes `model_config = ConfigDict(from_attributes=True)` (ALWAYS present)
- Includes primary key and any read-only/computed fields
- Includes nested relationship fields with forward references

```python
class Taksoni(TaksoniBase):
    model_config = ConfigDict(from_attributes=True)
    taksonin_nro: int
    heimo: Optional['Heimo'] = None
    viite: Optional['Viite'] = None
```

## Field Mapping Rules

### Required Fields

```python
name: str
count: int
created_at: datetime
```

### Optional Fields

```python
name: Optional[str] = None
count: Optional[int] = None
```

### Fields from ORM Model

The schema should mirror the ORM model exactly:

**ORM Model**:
```python
taksonin_nro: Mapped[int] = mapped_column(INTEGER(11), primary_key=True)
tieteellinen_nimi: Mapped[str] = mapped_column(String(255), nullable=False)
suku: Mapped[Optional[str]] = mapped_column(String(255))
```

**Schema**:
```python
class TaksoniBas(BaseModel):
    tieteellinen_nimi: str
    suku: Optional[str] = None

class Taksoni(TaksoniBase):
    model_config = ConfigDict(from_attributes=True)
    taksonin_nro: int
```

## Relationship Fields

Relationship fields use forward references (string quotes) to avoid circular imports:

```python
class Taksoni(TaksoniBase):
    model_config = ConfigDict(from_attributes=True)
    taksonin_nro: int
    heimo: Optional['Heimo'] = None
    viite: Optional['Viite'] = None
    taksoni_list: list['Taksoni'] = []
```

### Nested List Relationships

For one-to-many relationships, use `list[]`:

```python
class Heimo(HeimoBase):
    model_config = ConfigDict(from_attributes=True)
    jarjestysnumero: int
    taksoni_list: list['Taksoni'] = []  # Default to empty list
```

## Type Hints

| Python Type | Pydantic |
|---|---|
| int | int |
| str | str |
| bool | bool |
| float | float |
| datetime | datetime |
| date | date |
| Optional[int] | Optional[int] = None |
| list[int] | list[int] = [] |
| list['RelatedModel'] | list['RelatedModel'] = [] |

## Imports

Every schema file should have:

```python
from pydantic import BaseModel, ConfigDict
from typing import Optional, List
from datetime import datetime, date
```

## ConfigDict Settings

Always include `from_attributes=True`:

```python
model_config = ConfigDict(from_attributes=True)
```

This allows Pydantic to serialize from SQLAlchemy ORM objects directly using attribute access.

## Validation (Keep Minimal)

Avoid complex validation in schemas. Keep it simple:

```python
from pydantic import BaseModel, ConfigDict, Field

class Taksoni(TaksoniBase):
    model_config = ConfigDict(from_attributes=True)
    taksonin_nro: int
    tieteellinen_nimi: str = Field(..., min_length=1, max_length=255)
```

But prefer validation in the ORM model via constraints.

## Special Cases

### Composite Primary Keys

If the ORM model uses composite PKs, include all in the schema:

```python
class ListaTarkastajanimi(ListaTarkastajanimiBas):
    model_config = ConfigDict(from_attributes=True)
    id: int
    nimi: str
```

### No Relationships Loaded by Default

Due to `noload()` in `apply_filters()`, relationships return `None` by default in list endpoints:

```python
class Taksoni(TaksoniBase):
    model_config = ConfigDict(from_attributes=True)
    taksonin_nro: int
    heimo: Optional['Heimo'] = None  # Will be None in list responses
    viite: Optional['Viite'] = None   # Will be None in list responses
```

## Never

- ❌ Add fields that don't exist in the ORM model
- ❌ Use `Mapped[Optional[...]]` syntax (that's for ORM, use `Optional[...]` in Pydantic)
- ❌ Forget `model_config = ConfigDict(from_attributes=True)` on the final class
- ❌ Use bare relationship names without forward references: `heimo: Heimo` → use `heimo: Optional['Heimo']`
- ❌ Create schemas for non-API tables (legacy/test tables)
- ❌ Add complex validators that duplicate ORM logic
- ❌ Change field names from the ORM model

## If You Need to Create a New Schema

1. Check the corresponding ORM model in `app/models/<table>.py`
2. Create `app/schemas/<table>.py` with the three-class pattern
3. Include all columns from the ORM in the schema (matching name and type)
4. Use forward references for all relationship fields
5. Import the schema in `app/schemas/__init__.py`
6. Create an endpoint in `app/api/endpoints/<table>.py` that uses the schema

## Example: Complete Schema File

```python
from pydantic import BaseModel, ConfigDict
from typing import Optional

class TaksoniBase(BaseModel):
    tieteellinen_nimi: str
    suku: Optional[str] = None
    laji: Optional[str] = None
    suvun_auktori: Optional[str] = None
    lajin_auktori: Optional[str] = None

class TaksoniCreate(TaksoniBase):
    pass

class Taksoni(TaksoniBase):
    model_config = ConfigDict(from_attributes=True)
    taksonin_nro: int
    jarjestysnumero: Optional[int] = None
    viitenro: Optional[int] = None
    heimo: Optional['Heimo'] = None
    viite: Optional['Viite'] = None
```
